# クエスト/ストーリー設計案

この資料は、メインストーリークエストと各街のサブストーリークエストをゲームへ落とし込むための設計メモ。

前提:
- プレイヤーは冒険者ギルドを通じて世界を探索する。
- 世界の大きな謎は「AMC文明」「賢者の石」「マナ循環」「忘却」に集約する。
- NPCは悪役として消費せず、それぞれの生活、立場、誤解、願いを持たせる。
- サブストーリーは各街0〜2本。地域の文化、産業、信仰、マナ異常を見せる役割を持つ。
- クエストは後で実装しやすいように、`id / chapter / npc / start / objectives / flags / reward` で管理できる粒度にする。

他資料との整合方針:
- 地理、領名、Lv帯、地域導線は `AMC.md` を正本とする。
- 本編の最終ボスは `game_design_bible.md` と `AMC.md` に合わせて「賢者の石」。賢者像との精神リンク、purgeの書き換え、大魔法パージ習得を経て撃破する。
- マーチランド、アウターエッジ、忘却の遺跡群、アムネシアは賢者の石撃破後に続く一本道の忘却編として扱う。複数エンディングや隠し到達条件は置かず、必ず最終クリアまで到達できる構造にする。
- 賢者の石は古代から世界を維持していた装置ではなく、現代グラウバッハのエリクサー生物濃縮で生まれた高密度マナスライム。忘却は、AMC管理者権限を持つ知能アムネシアが担っていた危険参照の切断/存在消去/想起阻害機能が、パージ後の不安定化と現代のマナ濃度急上昇で暴走した問題として扱う。

## 1. 実装用クエスト仕様

### 1.1 クエスト種別

| 種別 | 用途 |
| --- | --- |
| `main` | メインストーリー。章単位で進行し、主要地域、機能、ボス、世界観を開放する。 |
| `sub` | 街ごとの連作サブストーリー。NPCや地域事情を掘る。 |
| `repeat` | ギルド掲示板などの反復依頼。既存の討伐/収集系。 |
| `hidden` | 特定の発言、石碑、踏破、ボス討伐などで始まる隠し依頼。 |

### 1.2 クエストデータ案

```js
{
  id: 'main_010_first_report',
  type: 'main',
  chapter: 1,
  title: 'はじめての報告',
  area: 'town',
  npc: 'ミラ',
  start: { visitedArea: 'town' },
  objectives: [
    { type: 'talk', npc: 'ミラ' },
    { type: 'kill', monster: 'slime', count: 3, area: 'field' },
    { type: 'talk', npc: 'ミラ' }
  ],
  unlocks: ['guild_basic', 'main_020_old_road'],
  rewards: { exp: 80, gem: 120, fame: 30 }
}
```

### 1.3 目的タイプ

| type | 内容 |
| --- | --- |
| `talk` | 指定NPCと会話する。 |
| `visit` | 指定エリアに入る。 |
| `kill` | 指定敵を指定数倒す。 |
| `boss` | 指定ボスを倒す。 |
| `collect` | 指定素材/ドロップ/クエストアイテムを集める。 |
| `inspect` | 石碑、施設、遺構、地点を調べる。 |
| `cast` | 指定魔法を詠唱する。 |
| `escort` | NPCを目的地まで連れていく。実装が重い場合は段階式 `visit + talk` で代用。 |
| `choice` | 短い返答や確認。ルート分岐には使わず、会話の温度差やNPC反応の差だけに留める。 |
| `sequence` | 複数地点を正しい順番で調べる。順番はNPCの生活情報や環境音から推理する。 |
| `sync` | 音、光、鐘、機関音などのタイミングに合わせて調査/詠唱する。失敗してもやり直せる。 |
| `route` | 危険区域を避けながら目的地へ進む。追跡、巡回、白い影などで緊張を作る。 |
| `defend` | 一定時間、灯、名簿、機械、NPCを守る。殲滅ではなく対象維持を目的にする。 |
| `decode` | 石碑、帳簿、歌詞、航路図などを照合する。戦闘前後の短い推理フェーズ。 |
| `memory` | 過去の会話や登録済みの名前を参照する。正解/不正解でルートは変えず、台詞と演出を変える。 |

### 1.4 報酬方針

- メイン: 次地域/機能開放、名声、基礎装備、重要魔法、重要ルーン、ワールド情報。
- サブ: 名声、gem、素材、見た目装備、街の施設割引、NPC好感度、地域限定ヒント。
- 隠し: レア見た目装備、称号、特殊会話、石碑ログ補完。

### 1.5 現行実装メモ

`index.html` では、ストーリークエストの土台として `talk / visit / kill / boss / inspect`、NPC頭上の `MAIN / SUB` マーカー、石碑調査マーカー、HUD目的表示、クエストログ、ストーリーログ、セーブ保存、称号パーツ報酬まで実装する。
現行実装はメイン17本、サブ60本、称号パーツは前半300/後半300の計600個を基準にする。
この段階ではルート分岐や隠しクリア条件は置かず、メインを追えば必ず次の目的へ進む。

称号報酬は、装備しなくても永続効果が重複する小ボーナスとして扱う。
ストーリー報酬で解放される称号は、討伐数やプレイ時間で自然解放される称号とは別に、キャラクターの旅の記録として残す。
称号の効果は `INT+1`、`HP+3`、`EXP獲得+0.12%`、`店割引+0.06%` のような微小値を積み上げる。装備中の二つ名は見た目と自己表現を担い、効果は解放済みパーツ全体から永続加算する。

現行UI:
- NPC頭上に `MAIN` / `SUB` を表示する。
- 石碑調査が現在目的の場合、対象石碑にも `MAIN` / `SUB` を表示する。
- HUD左側に進行中クエストを表示し、表示しきれない場合はその領域上のホイールでスクロールできる。
- ミニマップ上にクエスト目的地ガイドを表示する。目的地がミニマップ外なら端に方向矢印、範囲内なら地点リングを出す。討伐は対象敵の最寄り実体を優先し、未出現なら出現ゾーンを指す。ボスは対象エリア、会話/報告はNPCまたはギルド施設、調査は対象石碑/エリアを指す。
- `クエスト` スペルでクエストログを開く。タブは `クエスト`、`サブストーリークエスト`、`メインストーリークエスト`。
- `クエスト` タブは受注中の通常クエスト、`サブストーリークエスト` タブは未受注も含めて名称/発生場所/NPC、`メインストーリークエスト` タブは現在進行中/開始可/完了済みを表示する。
- メインストーリーの完了済み項目では、報酬ではなく「あらすじ」を読み返せるようにする。
- メニューの「ストーリーログ」で、進行中/開始可/完了/未解放を確認できる。

## 2. メインストーリーライン

### 全体テーマ

世界は古代AMC文明の崩壊後に成立した「新ユニ王国」であり、現在の文明はマナ循環、石碑、エリクサー経済、古代遺構の残骸に依存している。  
プレイヤーは地方の冒険者として始まり、各地の異常を解決するうちに、賢者の石事件が単独の災害ではなく、古代AMC文明、パージ、エリクサー封印、忘却の遺跡群へ続く大きな循環の表面化だと知る。

### 章ごとの体験変化

中盤以降は「新しい地域で調査してボス」を繰り返さない。各章に、短時間で理解できる専用の遊びを1つ入れる。  
専用ギミックは複雑なミニゲームにせず、既存の会話、調査、戦闘、移動、ログUIで表現する。

| 章 | 主体験 | 変化球 | 飽き対策の狙い |
| --- | --- | --- | --- |
| 4 ブラックウッド | 追跡と誤認の反転 | 敵を倒す前に痕跡を見る `route/decode` | 討伐依頼に見せて、観察が有利になる章にする。 |
| 5 レイクシャイア | 静かな調査と歌の照合 | 湖面、歌詞、宿帳を照合する `decode/sync` | 戦闘量を少し落とし、喪失感と推理でテンポを変える。 |
| 6 アクロス | 港の同時多発トラブル | 荷札、機関音、航路図を別々に集める `sequence` | 活気ある街で短い小目的を多くし、移動のリズムを上げる。 |
| 7 グラウバッハ | 工場の圧迫感と事故 | 稼働音、勤務表、白い雨を追う `sync/defend` | 時限ではなく「止めにくい」焦りを演出で作る。 |
| 8 アイゼンロア | 深部へ潜る緊張 | 地鳴りと測量図を合わせる `route/decode` | 縦方向の探索、環境音、封印解除でダンジョン感を変える。 |
| 9 パージ | 精神リンクと再戦 | 石碑条件を書き換える `memory/cast` | それまでの知識が戦闘前準備として効いている感触を出す。 |
| post 辺境 | 防衛と避難 | 名簿を守りながら白い影を退ける `defend/route` | 強敵討伐ではなく、人を残す緊張に切り替える。 |
| post 忘却 | プログラム書き換えと最終戦 | 旅の記録が戦闘演出へ割り込む `memory` | 一本道のまま、積み重ねが最後の処理修復に使われる体験にする。 |

## 3. サブストーリー設計方針

サブストーリーは、街ごとに完結する小話ではなく、地域ごとに連続する短編群として作る。  
各街のクエストは0〜2本に抑え、同じNPCが別の街や次の章で再登場してもよい。

分類:
- 生活: 食料、仕事、家族、移住、商売。
- 信仰: 教会、石碑、賢者伝承、循環思想。
- 産業: 農業、港湾、工業、採掘、開拓。
- 異常: 魔人、マナ濃度、忘却、記録欠落。
- 人間関係: 街同士の誤解、自治、王国への不信。

## 6. クリア導線と記録反映

基本方針:
- ルート分岐、複数エンディング、到達不能な隠しクリアは作らない。
- メイン進行を追えば、必ず賢者の石撃破、忘却編、アムネシア戦、エピローグまで到達する。
- サブストーリーはクリア条件ではなく、最終章の台詞、名前の壁、NPC後日談、エピローグの灯りを増やすための記録として扱う。

一本道の節目:
1. `main_710_purge_stone` で賢者の石を撃破し、急激なマナ災害を止める。
2. `post_800_border_forts` で境界砦の外縁異常を突破する。
3. `post_900_last_hold` でラストホールドと名前の壁へ到達する。
4. `post_910_amnesia` でアムネシアと対峙し、暴走を止めてプログラムを書き換える。
5. `post_990_record_cycle` で旅の記録を再編し、エピローグへ進む。

最終章で必ず明かすこと:
- 賢者の石は現代の濃縮実験で生まれた高密度マナスライムであり、古代パージの条件外に落ちたため暴走した。
- 忘却は賢者の石そのものの機能ではなく、AMC管理知能アムネシアが持つ危険参照の切断/存在消去/想起阻害機能が、パージ後の不安定化と現代のマナ情報過多で暴走して表面化したもの。
- アムネシアは破壊者ではなく、本来はAMC施設、スペル、危険思想を持つ生物を無力化/無害化するための管理知能だった。
- プレイヤーは「循環を止める」のではなく「忘れずに循環させる」方法を選ぶ。

サブストーリー反映:
- 未完了でも最終クリアは可能。
- 完了済みのサブストーリーは、名前の壁に灯る行数、最終戦前の短い声、エピローグの手紙/料理/歌/祈りとして反映する。
- 反映は報酬や分岐ではなく、プレイヤーが関わった生活が最後に見えるための演出にする。

## 8. 注意点

- サブストーリーのNPCは、悪意ではなく事情で動くようにする。
- メインで語る真相を、サブでは生活の違和感として先に見せる。

- 「賢者の石」は便利な資源ではなく、現代のエリクサー依存と古代パージの抜け穴が生んだ災厄として扱う。

## 9. ストーリー演出方針

### 9.1 複数NPCで厚みを出す

メイン/サブともに、依頼者だけで完結させない。  
1つのクエストに最低2人、重要クエストは3〜5人のNPCを絡める。

基本構造:

1. 依頼者が「表向きの問題」を提示する。
2. 別NPCが違う見方を話す。
3. 現地調査で、どちらの説明にも合わない違和感を見る。
4. 3人目のNPC、石碑、敵の行動、地形演出で真相の一部を出す。
5. 解決後、最初の台詞の意味が少し変わる。

例:
- 商人「積荷が盗まれた」
- 港湾長「盗難ではなく記録ミスだ」
- 船員「そもそもその積荷を積んだ記憶がない」
- 現地で空の木箱だけが残る
- 後半で「忘却は物そのものではなく、物や人に紐づく参照を危険対象として誤検出している」とわかる

### 9.2 伏線の種類

| 種類 | 使い方 |
| --- | --- |
| 生活伏線 | 甘すぎる麦、鳴らない鐘、名前が映らない湖面など、日常の小さな違和感。 |
| 台詞伏線 | NPCが何気なく言った言葉が、後で別の意味になる。 |
| UI伏線 | クエストログや地名、NPC名が一時的に変化する。やりすぎず重要場面だけ。 |
| 戦闘伏線 | 敵が逃げる、同じ場所を守る、プレイヤーを攻撃しない個体がいる。 |
| 石碑伏線 | 古代語や断片記録。真相を説明しすぎず、後から繋がる短文にする。 |
| 報酬伏線 | 見た目装備や素材名が後の真相を先取りする。 |

### 9.3 叙述トリックの扱い

プレイヤーをだますのではなく、NPCたちも誤解している構造にする。

使いやすい反転:
- 「盗まれた」ではなく「盗まれた記録だけが残った」
- 「魔人が襲った」ではなく「魔人だけが忘却の影響を受けにくく、助けようとしていた」
- 「賢者の石は救世主の遺物」ではなく「現代の濃縮実験と古代パージの抜け穴が生んだ災厄」
- 「アムネシアは世界を忘れさせる敵」ではなく「暴走したAMC管理知能」
- 「王国が隠した」ではなく「王国も忘れさせられていた」

### 9.4 感情の流れ

各地域で揺らす感情を変える。

| 地域 | 感情 |
| --- | --- |
| リンドフィー周辺 | 安心、少しの不安、旅立ち |
| アンバーフィールド領 | 豊かさへの感謝、不公平への苛立ち |
| 王都 | 威厳、違和感、制度への不信 |
| 黒森 | 恐怖、罪悪感、共感 |
| 湖地方 | 静けさ、喪失感、懐かしさ |
| アクロス | 活気、疑念、喪失の実害 |
| グラウバッハ | 便利さ、焦燥、倫理的な痛み |
| アイゼンロア | 誇り、疲労、掘り返す怖さ |
| 辺境 | 孤独、反抗、責任の重み |
| 忘却 | 寂しさ、怒り、受容、希望 |

### 9.5 中盤以降のテンポ設計

中盤以降はテーマが重くなるため、連続して「喪失」「罪」「事故」だけを置かない。  
各章に、短い達成、笑える生活会話、見た目報酬、景色の変化を混ぜ、プレイヤーが息継ぎできる場所を作る。

| 区間 | 重い要素 | 挟む軽い要素 | 目的 |
| --- | --- | --- | --- |
| 黒森 | 魔人差別、誤射、罪悪感 | 薬草の匂い当て、井戸の当番札、セラの過剰に丁寧な冗談 | 共生の生活感を先に見せ、説教臭さを避ける。 |
| 湖 | 名前欠落、霧の客 | リリの鼻歌更新、宿の部屋飾り、湖面写真風の記録 | 静かな章でも小さな変化を見つける楽しさを作る。 |
| アクロス | 名札消失、給金問題 | 市場の値切り会話、ピノの機械音メモ、船員の大げさな誓い | 活気でテンポを上げ、調査疲れを防ぐ。 |
| グラウバッハ | 労働事故、失業、賢者の石誕生 | 食堂献立、工場の休憩札、リクの皮肉、メルの小さな手書きメモ | 圧迫感の中にも人が暮らしている手触りを残す。 |
| アイゼンロア | 坑道事故、封印解除の怖さ | ピートの宝探し感、ガフの坑道迷信、結晶の見た目報酬 | 終盤前に探索のワクワクを取り戻す。 |
| 辺境 | 避難、名前の喪失 | 開拓地の地図書き足し、砦の炊き出し、ローガンの不器用な世話 | 終末感だけでなく、守る対象の生活を見せる。 |
| 忘却 | 削除、空白、最終戦 | モモの料理、エマの火番手順、ユンの下手な冗談 | 暗さの中で、最後に残すべき日常を明確にする。 |

配置ルール:
- 重い真相を出した直後は、必ず30秒以内に小さな生活反応を置く。
- 2つ続けて長い説明会話を置かない。説明の次は移動、短戦闘、環境変化、報酬のいずれかにする。
- 中盤以降のサブストーリー報酬は、数値よりも「街の表示変化」「NPC肩書き変化」「短い再訪台詞」を重視する。
- 戦闘が続く章では、戦闘前後に目的の性質を変える。例: 追跡 -> 短戦闘 -> 痕跡解読 -> ボス。
- 終盤は移動距離を伸ばしすぎない。精神リンク、索引、名簿などのUI/演出で密度を上げる。

- アムネシアへの選択肢は出さない。旅で集めた登録票、宛名、住所、歌、名札、勤務表、祈り、料理、研究ログが自動で索引として並び、アムネシアのプログラムが「無差別失効」から「危険参照の隔離と生活記録の保全」へ書き換わる。

## 12. 会話実装メモ

### 12.1 台詞段階

各クエストNPCは最低4段階の台詞を持つ。

| phase | 用途 |
| --- | --- |
| `before` | 受注前。噂や悩み。 |
| `start` | 受注時。目的提示。 |
| `progress` | 進行中。ヒント、別NPCへの誘導。 |
| `ready` | 報告可能。感情の変化。 |
| `done` | 完了後。後日談、伏線回収。 |

### 12.2 会話データ案

```js
{
  id: 'sub_miralake_010_reflection_name',
  speakers: ['フィネス', 'リリ', 'カレン', '霧の客'],
  lines: {
    before: {
      'フィネス': ['湖面は嘘をつきません。だから、嘘みたいなものが映ると困るんです。']
    },
    start: {
      'フィネス': ['名前だけが映らない人がいます。姿ではなく、名前だけです。調べてもらえますか。']
    },
    progress: {
      'リリ': ['その歌詞、意味は知らないの。でも、歌うと胸が痛くなる。'],
      'カレン': ['空欄の予約でも、部屋は空けておくものです。']
    },
    done: {
      '霧の客': ['私は誰かを待っていた。それだけで、今日は十分です。']
    }
  }
}
```

### 12.3 演出イベント案

| event | 内容 |
| --- | --- |
| `screen_desaturate` | 記憶/忘却に触れた時、一瞬だけ彩度を落とす。 |
| `area_title_blank` | エリア名表示が空白になる。終盤だけ使用。 |
| `npc_name_blank` | NPC名が空白になる。湖/忘却系で限定使用。 |
| `quest_log_glitch` | クエストログの一語が短時間だけ変わる。多用しない。 |
| `memory_overlay` | 複数NPCの短文を画面中央に薄く重ねる。精神リンク用。 |
| `wall_name_restore` | 名前の壁に1行戻る。サブストーリーで残した生活記録の可視化。 |
| `silent_bell` | 音を鳴らさず波紋だけ出す。信仰/忘却系。 |

### 12.4 実装優先度

最初に実装する厚めのクエスト:

1. `main_000_guild_card`
   - 名前/登録票伏線。
2. `sub_town_020_first_letter`
   - 手紙と名前の伏線。
3. `sub_mabel_010_sweet_wheat`
   - 生活異常。
4. `sub_fence_020_goblin_words`
   - 敵への見方を変える。
5. `main_030_watchtower_echo`
   - 古代記録の初提示。
6. `sub_univel_010_white_archive`
   - 王都の空白。
7. `sub_greenhollow_010_shared_well`
   - 魔人共生。
8. `sub_lasthold_020_names_wall`
   - 終盤の核。

この順番なら、序盤から終盤の真相まで同じモチーフ「名前/記録/忘却」が自然につながる。

## 14. メイン章ごとの詳細設計

### 14.1 第0章: 冒険者登録

テーマ:
- 世界に「名前を登録する」こと。
- プレイヤーが世界に記録される瞬間。

主なNPC:
- ミラ: 登録を担当する。明るいが、名前の扱いにだけ妙に厳しい。
- ティナ: 新人に親切な配達係。
- ガロ: まだ遠くから見ている街道番。

プレイヤーに思わせたいこと:
- ただのチュートリアル。
- ミラの「名前をなくす」は冗談。

本当の意味:
- この世界では名前が存在証明になる。
- ギルド登録は戦闘システムの入口であると同時に、忘却への対抗策。

イベント順:
1. ミラが登録を促す。
2. プレイヤー名を確認する。
3. 近郊で討伐。
4. ティナが手紙配達の話をする。
5. ミラが登録票を渡す。

会話メモ:
- ミラは必ず「名前」を2回以上言う。
- ティナは「返事」「配達」「届く」を強調する。
- ガロはまだ詳しく語らない。

演出:
- 登録完了時にクエストログへ「名前: 記録済み」。
- ここでは不穏にしすぎない。後で読み返すと怖いくらいがよい。

### 14.2 第1章: ハルフォード領と古街道

テーマ:
- 平和な地方に混じる小さな異常。
- 敵にも理由があること。

主要クエスト:
- `main_010_mabel_request`
- `main_020_old_road_signal`
- `main_030_watchtower_echo`

複数NPC構造:
- ティナ: 手紙が届かないことを心配する。
- ノル: 麦畑の異常を豊作と捉える。
- エル: ゴブリンの行動に素朴な疑問を持つ。
- ガロ: 古街道を危険視する。
- オルト: 古代遺物を価値ある商品として見る。

章の叙述トリック:
- ゴブリンが盗んでいるように見える。
- 実際はゴブリンも「名前が消える」現象に反応し、宛名を集めていた。
- 監視塔の声は怨霊ではなく、古代の巡回記録。

章ラスト:
- 監視塔の石碑で「名前を呼べ。戻れなくなる」を見る。
- プレイヤーは意味を理解できないが、ログには残る。

回収:
- 第9章の名前の壁で、宛名/登録票/巡回記録がすべて繋がる。

### 14.3 第2章: アンバーフィールド領と灌漑機構

テーマ:
- 豊かさは誰かの管理で成立している。
- 最適化は人間の生活感覚とずれる。

主要NPC:
- ロッシュ: 市場商人。数字を信じる。
- ミネット: 粉屋。味と手触りを信じる。
- セオ: 水番。順番と慣習を守る。
- フェン: ギルド巡回員。感情的対立を整理する。

プレイヤーに思わせたいこと:
- 水を盗んだ村がある。
- 市場の秤が壊れた。

本当の問題:
- 灌漑機構が収穫最大化のために記録と分配を自動補正している。
- 「不要」と判断された小さな畑や家の記録が欠け始めている。

重要な台詞:
- セオ:「水は量じゃない。順番だ」
- ミネット:「甘い麦はうれしい。でも、毎日食べるパンは甘すぎると疲れる」
- ロッシュ:「秤が嘘をついたら、商人は人を疑うしかなくなる」

演出:
- 水路沿いで、同じ水量なのに立札の数字だけ変わる。
- 灌漑機構ボス前の壁に「最適化」「廃棄」「再分配」。

章ラスト:
- 機構停止後、豊作は少し落ち着く。
- セオが「水の順番表」を手書きに戻す。

### 14.4 第3章: 王都と完全な記録

テーマ:
- 制度は強いが、制度自体も忘却されうる。
- 王国は敵ではなく、空白を抱えた大きな装置。

主要NPC:
- レーヴェン: 記録こそ秩序と信じる。
- アイラ: 書記の矛盾に気づく侍従。
- ユリア: 祈りの欠落を感じる。
- セラ: 科学/魔法研究の視点で空白を見る。

プレイヤーに思わせたいこと:
- 王国が都合の悪い歴史を隠している。

本当の問題:
- 隠蔽ではなく、隠蔽した記録ごと削られている。
- 王国は「完全な記録」を掲げるほど、空白に弱い。

演出:
- 書庫の白紙ページ。
- 教会の鐘が鳴らず、波紋だけが出る。
- 地下水路でNPC名が一瞬だけ空白になる。

章ラスト:
- レーヴェンは自分の記録観を曲げないが、初めて「未確認」と書く。
- セラが黒森/湖の異常へ興味を向ける。

### 14.5 第4-5章: ブラックウッドとレイクシャイア

テーマ:
- 恐れていた相手も、何かを守っていた。
- 名前を失っても、関係や歌や味は残る。

黒森側NPC:
- ニーノ: 危険から村を守る狩人。
- フェン: 過去の誤射を抱える元狩人。
- ラナ: 魔人から薬草を学んだ薬師。
- セラ: 魔人薬師。

湖側NPC:
- フィネス: 関係性の記録を研究する。
- リリ: 意味を知らず古い歌を歌う。
- カレン: 空欄の宿帳を守る。
- 霧の客: 名前を失った人物。

プレイヤーに思わせたいこと:
- 黒森は魔人問題。
- 湖は別種の幻想/霧の問題。

本当の問題:
- どちらも「名前が消えた後に、関係だけが残る」現象。
- 魔人はマナに近いため、忘却に対する耐性が少し違う。

演出:
- 魔人は最初攻撃せず逃げる個体を置く。
- 湖では名前欄が空白のNPCを出す。
- リリの歌詞が進行で1文字増える。

章ラスト:
- セラとフィネスの情報が繋がり、アクロス/グラウバッハ方面へ。

### 14.6 第6章: アクロスと消える積荷

テーマ:
- 記録の欠落は、経済と信頼を壊す。
- 世界は街道だけでなく、航路と帳簿で繋がっている。

主要NPC:
- ガルド: 商会頭。
- ネレ: 船長。
- ピノ: 整備士。
- アベル: 塩商。
- 無口な荷役: 名前だけが欠けた存在。

プレイヤーに思わせたいこと:
- 密輸か盗難か税の不正。

本当の問題:
- 物理的な積荷だけでなく、「誰が運んだか」の記録が抜けている。
- 飛空艇ドック跡のAMC輸送装置が、記録圧縮を行っていた。

演出:
- 港の掲示板に空白の名札。
- 飛空艇ドックで機関音だけが鳴る。
- ボス撃破後、名札の空白は戻らない。慰霊灯だけが増える。

### 14.7 第7章: グラウバッハと賢者の石

テーマ:
- 善意の技術が、人間を見なくなる瞬間。
- 効率化と救済は同じではない。

主要NPC:
- ヴァルター: 領主/技師。
- メル: 研究助手。
- リク: 失業工。
- ハンス: 現場を守る工場長。
- 工場の子ども: 親の勤務記録を探す。

プレイヤーに思わせたいこと:
- ヴァルターが冷酷な領主。
- 工場が人を搾取している。

本当の問題:
- ヴァルターは都市を救おうとしていた。
- エリクサー生物濃縮は、元が自然生物であるマナスライムを古代パージの対象外に置いたまま危険な密度へ押し上げていた。

演出:
- 賢者の石誕生時に白虹の粒子、反重力、時間加速を見せる。
- ヴァルターが石に触れようとして絶命する。
- プレイヤーも勝てない負けイベントを経験し、王都へ退避する。

章ラスト:
- 賢者の石は「便利な資源」ではなく、現代のエリクサー依存と古代パージの抜け穴が生んだ災厄だと認識が変わる。

### 14.8 第8章: アイゼンロアと採掘機構

テーマ:
- 掘るほど進む。掘るほど傷が開く。
- 資源の下には古代の判断が埋まっている。

主要NPC:
- ブラム: 鉱山監督。
- エッダ: 測量士。
- ガフ: 老坑夫。
- ピート: 独立鉱夫。
- セレネ: 環の教団の修道長。

プレイヤーに思わせたいこと:
- 採掘事故、労働問題、古代竜の巣。

本当の問題:
- AMC採掘機構は鉱脈ではなく、古代の制御部と封印設備を掘り当てかけていた。
- 地下にあるのは資源だけでなく、石碑プログラムを改写するための「中枢キー」。

演出:
- 坑道で低い歌のような振動。
- 壁面に結晶化した古代文字。
- 採掘機構ボス撃破後、地面ではなく画面全体が一瞬揺れる。

章ラスト:
- 中枢キーを入手。
- 王都の賢者像で、パージ改写を試みる導線が開く。

### 14.9 第9章: パージと賢者の石

テーマ:
- 古代の失敗を、現代の入力で書き換える。
- 世界を救う一撃は、世界を完全には治さない。

主要NPC:
- アンジェ: 賢者像を守る聖職者。
- 賢者: 休止していた古代AMC。
- レーヴェン: 王国の記録係。
- ミラ: 登録票の伏線を閉じる受付。

プレイヤーに思わせたいこと:
- 賢者像は信仰対象にすぎない。
- パージは古代の失われた禁忌魔法。

本当の問題:
- 賢者像は休止中の賢者本人であり、賢者の石誕生に反応して起動した。
- 旧パージは人工生命だけを対象にしており、自然生物由来の賢者の石には届かない。

演出:
- タイトルが一瞬「Another Mana Circulation」へ変化する。
- 精神リンク中、古代史は断片で見せ、説明しすぎない。
- パージ習得後、スペル一覧に物語限定大魔法として追加される。

章ラスト:
- パージで賢者の石を撃破する。
- マナ濃度上昇と忘却の遺跡群は残り、物語はマーチランドとアウターエッジへ続く。

### 14.10 ポストゲーム: マーチランドとアウターエッジ

テーマ:
- 王国の外側にも秩序がある。
- 本編で止めた災厄の奥に、古代パージと忘却の処理が残っている。

主要NPC:
- マーチランド辺境伯: 前線を守る責任者。
- ローガン: 古参兵。
- ノア: リベルタ代表。
- リタ: 開拓者。
- サージ: 外縁市の市場主。

プレイヤーに思わせたいこと:
- 辺境は危険で荒れている。
- 自治都市は王国に反抗している。

本当の問題:
- 辺境は忘却の進行を最前線で受けており、王国の手続きでは間に合わない。
- ノアたちは反逆ではなく、記録が消える前に生活を守っている。

演出:
- 壁の外側に白い影。
- 境界砦で敵名が「不明」になりやすい。
- リベルタでは王国式の肩書きではなく、名前だけで呼び合う。

章ラスト:
- ラストホールドへの道が開く。
- ノアが「名前で呼ぶ」ことをプレイヤーに頼む。

### 14.11 ポストゲーム: 忘却とアムネシア

テーマ:
- 忘れることは悪だけではないが、名前を奪う忘却は暴力。
- 世界を救うには、循環を止めるのではなく、記録の扱いを変える。

古代に忘却が必要だった理由:
- アムネシアは、AMCにおける管理者権限を持った古代知能。
- 石碑、人工生命、都市機構、転送門、スペル、パージ条件などは、名前や命令や対象指定が残っているだけで再起動する危険があった。
- アムネシアは、AMC由来の機構やスペルにおいて危険なもの、不要になったもの、失敗したものの参照を切り離し、必要なら存在そのものを消すことで安全を担保していた。
- AMC由来ではない生物に対しては、直接制御できない代わりに、マナを使って記憶に介入し、特定対象への想起を阻害することで無力化/無害化していた。
- 例: 危険思想を持つ人間から、その思想、合言葉、組織名、実行手順への想起を阻害し、行動に移せない状態へ落とす。
- 忘却は本来、生活や人名を無差別に消すためではなく、危険な参照、危険思想、危険手順を切るための管理者権限だった。
- 現在の忘却は、その判定と処理が壊れ、危険な参照と生活上の名前/宛名/所属を区別できなくなった状態。

人の記憶に効く理由:
- 忘却は脳や魂を直接破壊する力ではないが、マナを介して想起機能を強制的に乱す。
- この世界の人間の記憶は、肉体だけで完結せず、神経、マナ循環、名前、場所、道具、手紙、帳簿、歌、祈り、味などの手がかりが結びついて成立している。
- アムネシアの想起阻害は本来、危険思想、危険手順、危険対象への記憶経路だけを狙う制御だったが、壊れた現在では人のマナ循環に広く干渉し、名前や関係を思い出す経路まで一時的に遮断する。
- その結果、本人の脳内に記憶の断片が残っていても、想起の入口が塞がり、誰の記憶か、何に結びつく記憶かを引き出せなくなる。
- 症状は「脳から完全に消える」ではなく、「マナ性の記憶障害で呼び出せない」。知っている感触はあるのに名前が出ない、顔はわかるのに関係がわからない、歌えるのに意味がわからない、味や手つきだけ残る。
- 手紙、宿帳、勤務表、名前の壁など外部記録の欠落は、同じ失効処理が物理記録側にも作用した結果。人の記憶障害と外部記録の空白は、同じ現象の内側/外側の症状として扱う。
- 魔人は人間よりマナ循環への感覚が鋭く、口伝、薬草札、井戸の約束、身体感覚など複数の手がかりで記憶を支えるため、忘却の効き方が人間社会と少し違う。完全耐性ではないが、名前を失っても習慣や気配で補える場面がある。
- 高濃度マナ環境や精神リンクでは、想起機能と外部記録の境界が薄くなるため、忘却の影響が夢、幻聴、名前の空白、同じ夢を見る現象として表面化しやすい。

パージ後に残った理由:
- アムネシアは通常のAMC機構ではなく、管理者権限を持つ特殊な古代知能だった。
- 賢者と同じく、AMCのパージ対象として完全消去されず、権限と処理の一部だけを残した不安定な状態で残存した。
- パージ後は正常な監督系統を失い、危険判定、参照切断、想起阻害の境界が曖昧になった。

現代で暴走した理由:
- アムネシアは世界中のマナの流れを読み取り、そこから情報を得て演算、判定、処理を行っていた。
- 産業革命によるマナ濃度の急上昇、エリクサー経済の拡大、高濃度マナを持つエリクサースライムの出現によって、読み取る情報量が急激に増えた。
- グラウバッハのエリクサー生物濃縮と賢者の石事件は、アムネシア側から見ると異常な高密度マナ情報の連続入力だった。
- その結果、アムネシアのキャッシュがオーバーフローし、危険参照、生活記録、人名、思想、所属を区別する処理が壊れた。
- 忘却の遺跡群で起きる名前の空白、勤務表の欠落、同じ夢を見る現象は、アムネシアの処理異常が外部化したもの。

主要NPC:
- エマ: 火を守る。
- ユン: 名前を守る。
- モモ:味を守る。
- 無名の聖職者: 祈りの形を守る。
- アムネシア: AMC管理者権限を持つ古代知能。現在は暴走状態。

プレイヤーに思わせたいこと:
- アムネシアが忘却そのもの、または最終的な悪。

本当の問題:
- アムネシアは忘却そのものではなく、忘却機能を実行するAMC管理知能。
- 本来は危険なAMC機構、スペル、危険思想への参照を切り離す安全装置だった。
- 現在はマナ情報過多とパージ後の不安定化により、危険参照と生活記録を誤判定している。

最終クリア:
- アムネシアの暴走を止め、プログラムを書き換えて正常動作へ戻す。
- 書き換え後は、無差別な削除/想起阻害ではなく、危険参照の隔離と生活記録の索引保全を行う。
- 忘却の拡大は弱まるが、欠けた名前はすべて戻るわけではない。
- サブストーリーで残した名前/歌/味/祈りは、エピローグの声と灯りとして反映される。

ラスト台詞案:
- アムネシア:「私は忘却ではない。危険を呼び戻さぬため、参照を切る権限を与えられた」
- ユン:「順番じゃない。名前を呼ぶんだ」

## 19. 会話文の書き方ルール

### 19.1 基本

- 1回の吹き出しは短くする。
- 長い説明はNPC同士の食い違いで分割する。
- 世界観用語は初回だけ噛み砕く。
- NPCは自分の生活語彙で話す。研究者は記録、商人は信用、兵士は境界、料理人は味で語る。

### 19.2 避けたいこと

- NPCが設定資料をそのまま読む。
- 黒幕が全部説明する。
- サブクエストが単なるメインの補足説明になる。
- 魔人や自治都市を単純な善悪にする。
- プレイヤーだけが正しく、NPCが全員愚かに見える構図。

### 19.3 台詞の型

生活から入る:
```text
「麦が甘いんだ。いいことだと思うだろ？」
「でも、毎日食べるパンが甘いと、体が休まらない」
```

制度から入る:
```text
「記録にないものは、扱えません」
「では、記録から消えた人はどう扱うのですか」
```

反転前:
```text
「あいつらが盗んだに決まってる」
```

反転後:
```text
「盗んだんじゃない。残そうとしていたんだ」
```

終盤:
```text
「忘れないで、では足りない」
「呼ぶんだ。名前を」
```

## 20. 演出の強弱

### 20.1 序盤

- 演出は控えめ。
- 小物、手紙、登録票、灯などで伏線を置く。
- 怖さより「少し変だな」くらい。

### 20.2 中盤

- 白紙、名前空白、歌詞変化などを出す。
- NPC同士の意見対立を増やす。
- プレイヤーの判断が揺れる展開を入れる。

### 20.3 終盤

- UIや画面演出も物語に使う。
- 名前の壁、精神リンク、エリア名空白を使用。
- ただし乱発せず、忘却関連の重要場面に限定。

### 20.4 ポストゲーム

- 世界は完全には治らない。
- ただし記録方法が変わり、消えそうなものを残せるようになる。
- 高難易度ダンジョンやレア敵は「残った異常」として説明できる。

## 21. 実装前提の決定事項

### 21.1 仕様面

- クエストログには、最新会話だけでなく「記録断片」を残す。長文履歴ではなく、NPC名、場所、短い一文を保存する。
- 名前の壁はワールド上の施設として配置し、調べた時だけ専用UIを開く。普段から見える物体にして、生活空間の一部として扱う。
- 精神リンク演出はイベント専用にする。通常戦闘の状態異常や常用魔法にはせず、賢者像、賢者の石、アムネシア周辺の重要演出に限定する。
- NPC肩書き変更は保存する。例: フェンは「英雄」から「元狩人」、レーヴェンは「王室書記」から「未確認欄を作った書記」へ、会話上の呼ばれ方を変える。
- サブストーリー完了数は数値で大きく見せない。最終章では「聞こえる声」「表示される名前」「エピローグの一文」として反映し、到達条件やエンド分岐には使わない。

### 21.2 物語面

- アムネシアの口調は、感情の薄い管理者口調から始め、終盤だけ一人称の迷いを出す。敵意ではなく、管理者権限、参照、隔離、保全、削除、索引、失効、キャッシュ、オーバーフローという語彙を使う。
- オルランド王は象徴として遠ざけすぎず、人間味を短い台詞で出す。ただし政治を動かす万能人物にはしない。
- ミラは真相を知らない。登録票と名前を大事にするのは受付としての経験則で、終盤にその行為が意味を持っていたとわかる。
- 魔人が忘却に強い理由は、王国台帳ではなく口伝、薬草札、井戸の約束など複数の記録方法で生活しているため。完全耐性ではなく、消え方が人間社会と違う。
- 賢者の石撃破後も問題が残る理由は、賢者の石が急性のマナ災害で、忘却の遺跡群は古代パージ後に不安定化したアムネシアの管理者権限と危険参照の失効処理だから。原因が別階層にある。
- 最終クリア後の世界は完全復元しない。消えたものを全回収するのではなく、消えかけたものを「未確認」として留め、呼び戻す手順を得る。

### 21.3 最初に詳細確定するべきクエスト

1. `main_000_guild_card`
2. `sub_town_020_first_letter`
3. `sub_mabel_010_sweet_wheat`
4. `sub_fence_020_goblin_words`
5. `main_030_watchtower_echo`

この5本で、序盤のプレイヤー体験と最終章のテーマが決まる。  
ここを先に脚本レベルまで確定すると、その後の地域連作を同じ濃度で増やしやすい。

## 23. サブストーリー脚本チェック版: 生活感とパーソナリティ補強

目的:
- サブストーリーを「世界観説明」ではなく、NPCが普段の仕事、食事、癖、恐れ、誇りを持って生きている話として読めるようにする。
- すべて一本道クリア前提。選択で結末を分けず、調査順や完了済みクエストで台詞の温度だけを変える。
- 終盤の記録反映は、サブストーリーをクリアしないと到達できない条件ではなく、プレイヤーが見てきた生活が最後に声として戻る演出に留める。

### 23.1 ハルフォード/メイベル/フェンス・エンド

見せたい生活:
- ミラは受付として明るいが、毎朝いちばん先に登録台帳の角を揃える。名前が乱れることを本気で嫌がる。
- ティナは配達鞄の重さで一日の仕事量を測る。返事が来ない手紙を、失敗ではなく「まだ途中の配達」と考えている。
- ノルは麦の出来を見た目ではなく、噛んだ時の甘さと粉の手触りで判断する。
- ガロは街道番として、通った人数より帰ってこなかった人数を覚えている。
- エルは敵を「倒すもの」ではなく「何を持っていたか」で見る。

差し替え/追加台詞:
```text
ミラ:「台帳の端がそろってないと落ち着かないの。名前って、少し傾くだけで別の人みたいに見えるから」
ティナ:「返事が来ないと、配達鞄の片側だけ軽いままなんだ。軽いのに、ずっと肩が痛いの」
ノル:「豊作かどうかは、穂の数じゃない。朝のパンを二口目まで食べたくなるかだ」
ガロ:「門番は通した顔より、戻らなかった顔を覚える。覚えたくなくてもな」
エル:「ゴブリンの袋、石と釘と名前札が一緒に入ってた。宝物の分け方、ぼくたちと違うのかな」
```

演出:
- ミラの受付机に、まっすぐ並べた登録票と、角だけ擦り切れた古い台帳を置く。
- ティナの配達鞄は、未配達側だけ少し下がって見えるようにする。
- ノルの畑では、実りが豊かでもパンを食べる住人の反応が重い。
- ガロは会話中に街道の奥を何度も見る。プレイヤーを見ていない時間が長い。

見せたい生活:

- ミネットは粉屋として、祭りの甘いパンと毎日のパンを分けて考える。

```text

セオ:「水の順番表は、夜に書くんです。昼だと、みんな自分の畑のことしか言えなくなるから」
```

演出:


演出:
- 工場内の掲示は効率値だけでなく、食堂の献立や欠勤欄も見せる。
- メルのログには、数値の横に小さく労働者のあだ名が残っている。
- リクの閉じた工房には、未完成品ではなく、途中まで使い込まれた仕事道具を残す。
- ヴァルターの執務室は豪華にしすぎず、輸送費、雇用数、死亡事故数の紙束で圧迫する。

### 23.8 忘却の遺跡群/最終章

見せたい生活:
- エマは英雄的な灯守ではなく、火の番をしながら眠らないために日々の小さな手順を作っている。
- モモは料理で記憶を戻す便利役ではなく、名前が消えても味だけ先に残ることを知る生活者。
- ユンは名前を守る専門家だが、減った数を数える癖が自分を傷つけている。
- アムネシアは選択肢で裁く敵ではなく、AMC管理者権限を持ったまま不安定化した古代知能として描く。

差し替え/追加台詞:
```text
エマ:「火の番はね、英雄の仕事じゃない。薪を湿らせない、鍋を焦がさない、眠った人に毛布をかける。それだけ」
モモ:「この味、誰に教わったかは消えた。でも塩を入れる手つきだけ、まだ嘘をつかない」
ユン:「減った数を数えるのは得意になった。名前を呼ぶのは、まだ怖い」
アムネシア:「管理者権限、維持。危険参照、隔離。生活記録、危険参照として誤検出」
ミラ:「最初の登録票から始めよう。名前を一つ呼べたなら、次の名前も呼べる」
```

演出:
- ラストホールドでは、灯火だけでなく、干された布、使いかけの鍋、半分だけ読まれた本を配置する。
- 名前の壁は巨大な碑だけでなく、生活道具に貼られた名札が剥がれる形でも表現する。
- 最終戦前に選択肢は出さない。これまで見た名前、歌、帳簿、登録票、勤務表が短く連続表示され、プレイヤーが歩いた生活圏が索引として立ち上がる。

最終章の確認ポイント:
- プレイヤーは必ず `賢者の石撃破 -> 忘却編 -> アムネシア戦 -> エピローグ` へ進む。
- 未クリアのサブストーリーがあっても最終クリアは可能。
- クリア済みサブストーリーは、戦闘難度や到達条件ではなく、最後に聞こえる声、表示される名前、エピローグの一文として反映する。
- アムネシアを倒した後は「忘却が完全消滅」ではなく、「プログラムを書き換え、危険参照の隔離と生活記録の保全を両立できる状態」に変える。

## 25. 再訪会話とワールドステート補強

目的:
- クエスト完了後もNPCが同じ説明を繰り返すだけにならないよう、章進行に応じた短い再訪会話を用意する。
- 生活者の物語を「依頼完了」で止めず、街が少し変わる感触を出す。
- プレイヤーの進行は一本道のまま、見てきた人々が最後の索引演出に自然につながるようにする。

### 25.1 進行段階

再訪会話は、厳密な好感度ではなく章フラグで切り替える。

| 段階 | 条件 | 会話の温度 |
| --- | --- | --- |
| `before_local` | 地域メイン未完了 | 生活上の困りごと、噂、誤解。 |
| `after_local` | 地域メイン完了 | 問題は軽くなったが、余韻や後始末が残る。 |
| `after_sage_birth` | 賢者の石誕生後 | 各地がグラウバッハの異常を自分事として語る。 |
| `after_purge` | 賢者の石撃破後 | 急性災害は止まったが、忘却の不安が表面化する。 |
| `after_amnesia` | 最終クリア後 | 完全解決ではなく、残す方法を得た後の生活。 |

### 25.2 リンドフィー周辺

### 26.2 黒森: 撃つ前に見る

```text
目的: 襲撃跡を3つ調べる。
変化: 1つ目は血痕に見える赤い薬液、2つ目は引きずり跡に見える避難跡、3つ目は魔人の薬草袋。
```

- 王都の会話中心から、環境観察中心へ切り替わる。

実装方針:
- 手紙や物資は受け取らなくてもメイン進行できる。
- 受け取ると短い台詞、ログ、最終章の声が増える。
- 報酬は強い装備ではなく、見た目、称号、UI上の記録断片に寄せる。

### 26.11 プレイ時間の山谷

目安:
- 中盤の1章は、メインだけなら20〜35分で1つの大きな達成が見える。
- サブを含める場合でも、同じ種類の目的を3連続させない。
- 長いダンジョンの直後に長い会話を置く場合、会話前に街の変化や短い報酬表示を挟む。

山谷:
```text
黒森: 緊張 -> 反転 -> 共生の小さな安心
湖: 静けさ -> 照合 -> 半分だけ戻る歌
港: 活気 -> 混乱 -> 名札が戻る達成
工場: 圧迫 -> 事故 -> 退避と精神リンクへの不安
鉱山: 探索 -> 発見 -> パージ準備の高揚
賢者の石: 再戦 -> 撃破 -> まだ終わらない違和感
辺境: 勝利後の不安 -> 防衛 -> 名前を守る目的の明確化
忘却: 喪失 -> 索引化 -> 完全ではない希望
```

チェック項目:
- 連続する2章で同じ「調査してボス」構成になっていないか。
- 長い説明の直前に、プレイヤーが自分で見た証拠があるか。
- 重要NPCが説明役だけにならず、生活者としての反応を持っているか。
- 章クリア後に、街の表示、NPC肩書き、BGM/環境音、ログのいずれかが変わるか。
- 中盤以降も序盤NPCからの便りがあり、旅の連続感があるか。

### 28.6 終盤の整理

- ルート分岐、上位エンド、隠し到達条件は置かない。
- メインを追えば `最終区画 -> アムネシア暴走停止 -> プログラム書き換え -> 本編クリア` へ必ず到達する。
- 実装上は `main_016_final_sector_rewrite` 完了時に `main_clear` と `amnesia_rewritten` を立て、ストーリーログにも本編クリア状態を表示する。
- `main_017_postgame_patrol` は `postgame_patrol_checked` を立てる後日談巡回であり、クリア判定を置き換えない。
- ポストゲームは「より正しい結末」ではなく、書き換え後の安全装置を点検する通常運用として扱う。
- `recordCore` は世界に必要な巨大記録庫ではなく、アムネシアへ命令/反証記録を渡すためのAMC管理中継核として扱う。
- サブストーリーはクリア条件ではなく、生活記録を増やすための横道。未クリアでも本編クリアは可能。
- 終盤で使う生活記録は、手紙、舞台の合図、花札、門の点検音、寝台札、灯り、地図の余白など、キャラクターの仕事や暮らしに紐づける。
